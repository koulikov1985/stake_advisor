# Fix Render Deployment Failures - Complete Guide

## Issue

Your Render deployment is failing with "Exited with status 1" errors. The frontend builds successfully, but the backend service is failing to start.

## Root Cause

The render.yaml file has several environment variables marked as `sync: false`, which means they are **NOT automatically set**. You need to manually add them in the Render dashboard.

## Required Environment Variables

### 1. Backend Service (stake-advisor)

Go to your Render dashboard → Select "stake-advisor" service → Environment tab → Add these variables:

#### Critical (Required for startup):

```
ADMIN_EMAIL=your-email@example.com
ADMIN_PASSWORD=choose-a-strong-password
```

#### Optional (Can add later):

```
REDIS_URL=redis://[your-redis-url]  (Only if you plan to use Celery workers)
PADDLE_API_KEY=pdl_xxx  (Only if using Paddle payments)
PADDLE_WEBHOOK_SECRET=whsec_xxx  (Only if using Paddle payments)
SMTP_HOST=smtp.your-provider.com  (Only if sending emails)
SMTP_PORT=587
SMTP_USER=your-smtp-user
SMTP_PASSWORD=your-smtp-password
```

### 2. Environment Variables Already Set

These should already be configured (verify they exist):

✅ **DATABASE_URL** - Auto-set from your database
✅ **SECRET_KEY** - Auto-generated by Render
✅ **PYTHON_VERSION** - Set to 3.11
✅ **VITE_API_URL** - Set for frontend

## Step-by-Step Fix Instructions

### Step 1: Add Required Environment Variables

1. Go to https://dashboard.render.com
2. Click on your **stake-advisor** service (backend)
3. Click **Environment** in the left sidebar
4. Click **Add Environment Variable**
5. Add these two variables:
   ```
   Key: ADMIN_EMAIL
   Value: admin@sharkpokerclub.com

   Key: ADMIN_PASSWORD
   Value: [choose a secure password - save this somewhere safe!]
   ```
6. Click **Save Changes**

### Step 2: Verify Database Connection

1. In your Render dashboard, go to the **stake-advisor** service
2. Click **Environment** tab
3. Find **DATABASE_URL** - verify it exists and looks like:
   ```
   postgresql://username:password@host/database
   ```
4. If missing, click **Add Environment Variable** and select:
   - Key: `DATABASE_URL`
   - Value: Select "From Database" → `license-keys-db`

### Step 3: Check Deployment Logs

After adding the environment variables, Render will auto-deploy. To check if it's working:

1. Go to your **stake-advisor** service
2. Click **Logs** tab
3. Look for these success messages:
   ```
   ✅ "INFO:     Application startup complete"
   ✅ "Creating admin user with email: admin@sharkpokerclub.com"
   ✅ "Admin user created/updated"
   ```

### Step 4: Fix Common Errors

#### Error: "ADMIN_EMAIL is not set"
- **Solution**: Add ADMIN_EMAIL environment variable (Step 1)

#### Error: "ADMIN_PASSWORD is not set"
- **Solution**: Add ADMIN_PASSWORD environment variable (Step 1)

#### Error: "Could not connect to database"
- **Solution**: Verify DATABASE_URL is set correctly (Step 2)
- Make sure the database "license-keys-db" exists and is running

#### Error: "Alembic migration failed"
- **Solution**: Database might be locked or have schema issues
- Try: Go to Shell tab in Render and run: `alembic downgrade -1` then `alembic upgrade head`

### Step 5: Test Your Deployment

Once deployment succeeds:

1. Backend health check:
   ```
   https://stake-advisor.onrender.com/health
   ```
   Should return: `{"status": "ok"}`

2. Backend API docs:
   ```
   https://stake-advisor.onrender.com/docs
   ```

3. Frontend:
   ```
   https://sharkpokerclub.com
   ```

## Troubleshooting

### Deployment Still Failing?

1. **Check Python Version**:
   - Your render.yaml specifies Python 3.11
   - Verify in Environment: `PYTHON_VERSION = 3.11`

2. **Check Build Logs**:
   - Go to your service → Logs tab
   - Look for specific error messages
   - Common issues:
     - Missing dependencies (check requirements.txt)
     - Import errors (check all files committed)
     - Database connection timeout

3. **Manual Deploy**:
   - Go to service → Manual Deploy → Deploy latest commit
   - Watch the logs in real-time

### Need to Access Logs?

```bash
# View recent logs
1. Go to Render dashboard
2. Click on "stake-advisor" service
3. Click "Logs" tab
4. Scroll to bottom for most recent errors
```

## Important Notes

### About REDIS_URL
- **You don't need Redis** unless you plan to use Celery background workers
- The backend will work fine without it
- Only add REDIS_URL if you specifically need async task processing

### About Email (SMTP)
- **Email is optional** for basic functionality
- Only add SMTP variables if you need to send emails (password reset, etc.)

### About Payments
- **Paddle/Stripe are optional** for initial deployment
- Add payment variables later when you're ready to process payments
